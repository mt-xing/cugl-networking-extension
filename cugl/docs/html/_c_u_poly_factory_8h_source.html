2 // CUPolyFactory.java
3 //
4 // This module provides a convenient way to generate simple polygons, like lines
5 // or circles. It is lighter weight than the other factory classes because it
6 // does not separate the calculation step from the materialization step. That
7 // is because all of its calculations are very short and do not need to be
8 // factored into a separate thread.
9 //
10 // Because math objects are intended to be on the stack, we do not provide
11 // any shared pointer support in this class.
12 //
13 // This implementation is largely inspired by the LibGDX implementation from
14 // Nicolas Gramlich, Eric Spits, Thomas Cate, and Nathan Sweet.
