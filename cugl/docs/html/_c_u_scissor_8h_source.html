2 // CUScissor.h
3 // Cornell University Game Library (CUGL)
4 //
5 // This module provides support for a scissor mask that supports rotation and
6 // other transforms. A scissor mask is a rectangular region that whose size
7 // is defined by the extent attribute. The associated transform transforms
8 // this rectangle about its center.
9 //
10 // This module is based on the NVGpaint datatype from nanovg by Mikko Mononen
11 // (memon@inside.org). It has been modified to support the CUGL framework.
12 //
13 // This class uses our standard shared-pointer architecture.
14 //
15 // 1. The constructor does not perform any initialization; it just sets all
16 // attributes to their defaults.
17 //
18 // 2. All initialization takes place via init methods, which can fail if an
19 // object is initialized more than once.
20 //
21 // 3. All allocation takes place via static constructors which return a shared
22 // pointer.
23 //
24 // With that said, this class looks very similar to the classes in the math
25 // package in that has many methods that assume the object is on the stack and
26 // not in a pointer. That is because we often want to transform these objects
27 // with math classes, but we still want shared pointer support for SpriteBatch
28 // management. The result is a class with a bit of a hybrid feel.
