5 // This class is represents a two-pole IIR filter. For second-degree filters,
6 // it is significantly more performant than IIRFilter.
7 //
8 // This class supports vector optimizations for SSE and Neon 64. In timed
9 // simulations, these optimizations provide at least a 3-4x performance
10 // increase (and in isolated cases, much higher). These optimizations make
11 // use of the matrix precomputation outlined in "Implementation of Recursive
12 // Digital Filters into Vector SIMD DSP Architectures"
13 //
14 // https://pdfs.semanticscholar.org/d150/a3f75dc033916f14029cd9101a8ea1d050bb.pdf
15 //
16 // The algorithm in this paper performs extremely well in our tests, and even
17 // out-performs Apple's Acceleration library. However, our implementation is
18 // limited to 128-bit words as 256-bit (e.g. AVX) and higher show no significant
19 // increase in performance.
20 //
21 // For performance reasons, this class does not have a (virtualized) subclass
22 // relationship with other IIR or FIR filters. However, the signature of the
23 // the calculation and coefficient methods has been standardized so that it
24 // can support templated polymorphism.
25 //
26 // This class is NOT THREAD SAFE. This is by design, for performance reasons.
27 // External locking may be required when the filter is shared between multiple
28 // threads (such as between an audio thread and the main thread).
