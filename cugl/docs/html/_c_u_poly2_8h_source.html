3 // Cornell University Game Library (CUGL)
4 //
5 // This module provides a class that represents a simple polygon. The purpose
6 // of this class is to separate the geometry (and math) of a polygon from the
7 // rendering data of a pipeline. It is one of the most important classes for
8 // 2D game design in all of CUGL.
9 //
10 // Polygons all have a corresponding geometry. If they are implicit, they
11 // cannot be drawn, but can be used for geometric calculation. Otherwise,
12 // the polygon has a mesh defined by a set of vertices. This class is
13 // intentionally (based on experience in previous semesters) lightweight.
14 // There is no verification that indices are properly defined. It is up to
15 // the user to verify and specify the components. If you need help with
16 // triangulation or path extrusion, use one the the related factory classes.
17 //
18 // Because math objects are intended to be on the stack, we do not provide
19 // any shared pointer support in this class.
