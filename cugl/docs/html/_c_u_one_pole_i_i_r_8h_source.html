5 // This class is represents a one-pole IIR filter. This is the standard class
6 // for implementing first order lowpass filters. For first-order filters,
7 // it is significantly more performant than IIRFilter.
8 //
9 // This class supports vector optimizations for SSE and Neon 64. In timed
10 // simulations, these optimizations provide at least a 3-4x performance
11 // increase (and in isolated cases, much higher). These optimizations make
12 // use of the matrix precomputation outlined in "Implementation of Recursive
13 // Digital Filters into Vector SIMD DSP Architectures"
14 //
15 // https://pdfs.semanticscholar.org/d150/a3f75dc033916f14029cd9101a8ea1d050bb.pdf
16 //
17 // The algorithm in this paper performs extremely well in our tests, and even
18 // out-performs Apple's Acceleration library. However, our implementation is
19 // limited to 128-bit words as 256-bit (e.g. AVX) and higher show no significant
20 // increase in performance.
21 //
22 // For performance reasons, this class does not have a (virtualized) subclass
23 // relationship with other IIR or FIR filters. However, the signature of the
24 // the calculation and coefficient methods has been standardized so that it
25 // can support templated polymorphism.
26 //
27 // This class is NOT THREAD SAFE. This is by design, for performance reasons.
28 // External locking may be required when the filter is shared between multiple
29 // threads (such as between an audio thread and the main thread).
