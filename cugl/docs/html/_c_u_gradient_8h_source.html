2 // CUGradient.h
3 // Cornell University Game Library (CUGL)
4 //
5 // This module provides support for a two-color gradient. It uses a box
6 // gradient definition to support both linear gradients and radial gradients
7 // in addition to box gradients.
8 //
9 // This module is based on he NVGpaint datatype from nanovg by Mikko Mononen
10 // (memon@inside.org). It has been modified to support the CUGL framework.
11 //
12 // This class uses our standard shared-pointer architecture.
13 //
14 // 1. The constructor does not perform any initialization; it just sets all
15 // attributes to their defaults.
16 //
17 // 2. All initialization takes place via init methods, which can fail if an
18 // object is initialized more than once.
19 //
20 // 3. All allocation takes place via static constructors which return a shared
21 // pointer.
22 //
23 // With that said, this class looks very similar to the classes in the math
24 // package in that has many methods that assume the object is on the stack and
25 // not in a pointer. That is because we often want to transform these objects
26 // with math classes, but we still want shared pointer support for SpriteBatch
27 // management. The result is a class with a bit of a hybrid feel.
