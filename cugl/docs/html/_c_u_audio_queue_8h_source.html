5 // This module provides a queue for seamless audio playback. You can use
6 // this queue to dynamically switch between music loops with no break in
7 // the middle. Typically there is only one audio queue, but it is possible
8 // to have as many as you want.
9 //
10 // Music queues are owned by the audio engine. Shutting down that engine
11 // will shut down an associated audio queue as well.
