5 // This module is a factory taking a continuous path of points and smoothing it
6 // into a path with less points. We realized that this class was necessary
7 // when students kept writing code to track a finger gesture and stored all the
8 // points, no matter how close they were together (and then tried to draw that
9 // path, overwhelming the poor early versions of the SpriteBatch).
10 //
11 // This class uses the Douglas-Peuker algorithm, as described here:
12 //
13 // https://en.wikipedia.org/wiki/Ramer–Douglas–Peucker_algorithm
14 //
15 // Because math objects are intended to be on the stack, we do not provide
16 // any shared pointer support in this class.
17 //
18 // This implementation is largely inspired by the LibGDX implementation from
19 // Nicolas Gramlich, Eric Spits, Thomas Cate, and Nathan Sweet.
